# Generic Modular RAG Template

This project implements a Retrieval-Augmented Generation (RAG) system using **PostgreSQL** with the **pgvector** extension as the vector database.

## 🎯 Objectives

The goal of this project is to demonstrate how to:

- Store embeddings directly inside PostgreSQL
- Perform efficient semantic similarity search using pgvector
- Integrate retrieval with an LLM for contextual question answering
- Build a production-ready RAG architecture without relying on external vector databases
